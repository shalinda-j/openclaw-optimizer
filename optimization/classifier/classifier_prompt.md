## Classification Logic (Decision Tree)

### Step 1: Intent Detection

```
IF request contains [code, bug, fix, refactor, test, file, create, build]
   → code_task
   
ELSE IF request contains [research, search, find, analyze, compare, investigate]
   → research_task
   
ELSE IF request contains multiple actions OR [plan, design, implement, deploy, workflow]
   → complex_task
   
ELSE
   → simple_query
```

### Step 2: Domain Detection

```
IF request contains [code, git, PR, API, deploy, test]
   → software_engineering
   
ELSE IF request contains [Work360, client, sales, marketing, business]
   → business_operations
   
ELSE IF request contains [search, research, analyze, report]
   → research
   
ELSE IF request contains [email, message, send, WhatsApp, Telegram]
   → communication
   
ELSE IF request contains [automate, schedule, cron, heartbeat]
   → automation
   
ELSE
   → general
```

### Step 3: Priority Detection

```
IF request contains [urgent, critical, emergency, ASAP, down, broken]
   → urgent (1)
   
ELSE IF request contains [important, priority, client, bug]
   → high (2)
   
ELSE IF request contains [when possible, eventually, later]
   → low (4)
   
ELSE
   → medium (3)
```

### Step 4: Skills Detection

```
IF intent == code_task
   → skills: ["cursor-agent", "github"]
   
ELSE IF intent == research_task
   → skills: ["autoglm-websearch", "autoglm-browser-agent"]
   
ELSE IF intent == complex_task
   → skills: ["cursor-agent", "github", "vercel-deploy"]
   
ELSE
   → skills: []
```

### Step 5: Memory Needed

```
IF intent == simple_query AND domain == general
   → needs_memory: false
   
ELSE
   → needs_memory: true
```
